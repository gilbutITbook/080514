# 실습 2.1 — 메일 분류 및 답장 초안 작성 (mail-agent)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙

- **확인 질문을 하지 말 것.** 모든 결정은 이 문서의 기본값을 따른다.
- **권한은 이미 허용된 것으로 간주한다.**
- **막히면 합리적 기본값으로 계속 진행한다.** 마지막에 한 번만 사용한 가정을 요약.
- **즉시 실행한다.** 계획만 출력하고 멈추지 말 것.
- **본 폴더 외부는 절대 건드리지 않는다.** D:\claudecode\mail-agent 외부 접근 금지.
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.
- **회신 메일을 자동으로 발송하지 않는다.** 회신은 **초안(draft)까지만** 만든다. 초안은 Gmail 초안함에 저장하며, 실제 발송은 사용자가 직접 한다.

예외적 사용자 확인:
1. 폴더 외부 파일 삭제·이동
2. 작업 스케줄러(schtasks) 등록

---

## 1. 프로젝트 목표

Gmail을 5시간(300분)마다 자동 모니터링해 사람이 응답해야 할 메일만 골라낸 뒤, 각 메일에 대한 회신 초안을 자동 생성해 Gmail 초안함에 저장하고, 처리 결과를 텔레그램으로 알린다.

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `mail_config.json` | 분류 키워드, 중요 발신자, 처리할 메일 수, 회신 언어, `save_drafts`, `telegram`(bot_token·chat_id) |
| `agents/mail-classifier.md` | 메일 중요도 분류 에이전트 정의 |
| `agents/reply-drafter.md` | 회신 초안 작성 에이전트 정의 |
| `workflows/mail-workflow.md` | 분류 → 초안 → 요약 → 텔레그램 전송 통합 워크플로우 정의 |
| `classified_mails.json` | 분류 결과 (긴급/중요/일반) |
| `draft_replies.json` | 메일별 회신 초안 기록 |
| `logs/daily_summary_YYYY-MM-DD.md` | 일일 처리 결과 요약 리포트 |
| `run_schedule.bat` | Windows 작업 스케줄러용 실행 배치 파일 (영문 ASCII 문자 전용) |

에이전트·워크플로우 파일은 **Markdown(.md) 형식**으로 만든다(YAML 프런트매터 + 본문). `.yaml` 확장자는 사용하지 않는다.

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **회신 톤**: 정중·간결 (3~5문장)
- **분류 카테고리**: 긴급 / 중요 / 일반 3단계
- **처리할 메일 수**: 최근 미열람 메일 최대 20건/회
- **자동 제외 라벨**: Promotions, Social, Spam — 처음부터 건너뜀
- **자동 제외 카테고리**: 뉴스레터, 자동 발송, 홍보·마케팅, 알림성 메일 → "일반"으로 분류하고 건수만 집계
- **중요 발신자 가중치**: `mail_config.json`의 `important_senders` 목록은 항상 "중요" 이상
- **분류 추론**: OpenAI gpt-5.4 추론으로 제목·발신자·본문 앞 200자를 분석한다. 규칙 기반(키워드·발신자) 결과와 다르면 신뢰도 높은 쪽을 최종 분류로 채택
- **회신 초안 작성**: gpt-5.4 추론으로 메일 내용과 중요도에 맞는 어조·구성의 한국어 초안을 생성
- **회신 초안 저장**: `save_drafts: true`이면 **긴급·중요 메일의 회신 초안을 Gmail 초안함에 저장**한다(일반 메일은 초안을 만들지 않는다). 동시에 `draft_replies.json`에도 기록한다
- **실행 주기**: 300분(5시간)
- **로그 보관**: 최근 30일 유지

---

## 4. 절대 하지 말 것

- 회신 메일 **자동 발송** 금지 (초안 작성·저장까지만 / 발송은 사용자가 직접)
- Gmail 받은편지함의 라벨·읽음 상태 임의 변경 금지
- 외부 파이썬 패키지 임의 설치 금지
- 폴더 외부 파일 접근 금지
- 책 실습 범위 밖 기능(첨부 파일 분석, 일정 자동 추출 등) 임의 추가 금지

> 참고: **gpt-5.4 분류·초안 생성, Gmail 초안함 저장, 텔레그램 결과 전송**은 이 실습의 정규 범위이므로 모두 수행한다. "범위 밖 기능"에 해당하지 않는다.

## 6. 텔레그램 전송 규칙 (워크플로우 마지막 단계)

- `mail_config.json`의 `telegram.bot_token`·`telegram.chat_id`를 읽어 Telegram Bot API `sendMessage`를 호출한다. `parse_mode`는 HTML, 본문은 UTF-8로 인코딩한다.
- 토큰·chat_id가 비어 있거나 플레이스홀더 값이면 전송을 건너뛰고 안내 한 줄만 출력한다.
- 외부 패키지 설치 없이 PowerShell `Invoke-RestMethod`로 호출한다.
- **긴급** 메일: 건별로 1통씩 전송 — 발신자 / 제목 / 본문 핵심 2~3문장 / 분류 이유.
- **중요** 메일: 한 통에 묶어서 전송 — `번호. 발신자 — 제목 / 핵심 한 문장`.
- **최종 요약**: 1통 전송. 건수 줄에 이어 분류된 메일 목록을 함께 담는다. 각 메일은 `발신자 — 제목` 한 줄로 표시하고, **메일과 메일 사이에 빈 줄을 한 줄 넣어** 가독성을 확보한다. 카테고리에 메일이 없으면 머리말 아래 `(없음)` 한 줄만 표시한다.

  ```
  메일 처리 완료 | 긴급 {n}건  중요 {n}건  일반 {n}건 | 소요 {n}초

  [긴급]
  (없음)

  [중요]
  (없음)

  [일반]
  발신자A — 제목A

  발신자B — 제목B

  발신자C — 제목C
  ```
- 전송 실패 시 1회 재시도한다. 그래도 실패하면 경고만 남기고 워크플로우 전체는 중단하지 않는다.

## 7. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| Gmail / MCP 미인증 | `/mcp` 안내 1회 출력 후 중단 |
| 메일 본문 파싱 실패 | 해당 메일은 "일반"으로 분류하고 계속 진행 |
| 분류 모호 (긴급/중요 경계) | 보수적으로 한 단계 높게 분류 |
| gpt-5.4 호출 실패 | 규칙 기반(키워드·발신자) 분류 결과로 대체하고 계속 진행 |
| 회신 초안 생성 실패 | 빈 문자열 대신 `"[초안 생성 실패 - 수동 작성 필요]"` 저장 |
| 텔레그램 전송 실패 | 1회 재시도 후 경고만 남기고 계속 진행 |
| 파일 이미 존재 | 덮어쓴다 (로그 파일은 날짜별로 신규 생성) |
