# 실습 1.1 — 가족 여행 계획 설계 (family-travel-planner)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙 (반드시 지킬 것)

- **확인 질문을 하지 말 것.** 사용자에게 "진행할까요?", "어떤 방식으로 할까요?", "이 파일을 만들어도 될까요?" 류의 질문을 하지 않는다. 모든 결정은 이 문서의 기본값을 따른다.
- **권한은 이미 허용된 것으로 간주한다.** `.claude/settings.json`에 허용된 도구 사용은 사용자에게 다시 묻지 않는다.
- **막히면 멈추지 말고 합리적 기본값으로 계속 진행한다.** 정보가 부족하면 가장 그럴듯한 값으로 채우고, 작업을 끝까지 마친 뒤 마지막에 한 번만 "사용한 가정"을 요약 보고한다.
- **계획만 출력하고 멈추지 말 것.** 사용자가 "~~를 만들어줘"라고 요청하면 즉시 만들고, 끝난 뒤 결과를 보고한다.
- **선택지를 늘리지 말 것.** "옵션 A / 옵션 B 중 어느 쪽을 원하세요?"라고 묻지 않는다. 기본값을 골라 진행한 뒤, 필요하면 사용자가 수정 요청을 한다.
- **본 폴더 외부는 절대 건드리지 않는다.** D:\claudecode\family-travel-planner 외부의 파일을 읽거나 수정하지 않는다.
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.

예외적으로 사용자 확인이 필요한 경우는 단 두 가지뿐이다:
1. 이 폴더 **외부**의 파일을 삭제·이동해야 하는 경우
2. 사용자 명의의 외부 메시지(메일·메신저)를 실제로 발송해야 하는 경우 (이번 실습에는 해당 없음)

---

## 1. 프로젝트 목표

가족 여행 정보(여행지·기간·예산·구성·스타일)를 입력하면 다음을 자동 생성하는 오픈클로 워크플로우를 만든다.

- 일자별 관광 코스
- 예산 맞춤형 숙소 추천
- 예산 검증·자동 조율
- 최종 여행 계획서 (마크다운)

최종 워크플로우 이름: `family-travel-workflow`

---

## 2. 산출물 명세 (이 파일들이 만들어져야 작업 완료)

폴더는 `family-travel-planner/` 안에서만 작업한다.

| 파일 | 내용 |
|------|------|
| `travel_input.py` | 여행 조건을 받아 `travel_input.json`을 생성하는 파이썬 스크립트 |
| `travel_input.json` | 여행지·기간·예산·구성·스타일이 담긴 입력 데이터 |
| `agents/travel-course-agent.md` | 여행 코스 설계 에이전트 정의 |
| `travel_course.json` | 일자별 관광 코스 (오전/오후/저녁, 이동시간, 식당, 비용 포함) |
| `agents/accommodation-agent.md` | 숙소 추천 에이전트 정의 |
| `accommodation.json` | 가격대별 3단계 숙소 옵션 |
| `budget_summary.json` | 예산 검증 결과 및 조정안 |
| `travel_plan.md` | 최종 가족 여행 계획서 (마크다운) |
| `workflows/family-travel-workflow.md` | 통합 오픈클로 워크플로우 정의 |

---

## 3. 사전 결정된 기본값 (질문 금지)

다음 항목은 사용자에게 묻지 않고 아래 값을 사용한다.

- **언어**: 한국어 (모든 출력·주석·문서)
- **인코딩**: UTF-8
- **JSON 들여쓰기**: 2-space
- **날짜 형식**: `YYYY-MM-DD`
- **통화 단위**: 원(₩), 정수
- **숙박비 비중**: 총 예산의 40%
- **숙소 추천 옵션 수**: 절약형 / 표준형 / 프리미엄 3단계
- **예산 초과 시 자동 조정 순서**: ① 숙소 등급 하향 → ② 유료 관광지를 무료 대안으로 교체 → ③ 외식 횟수 축소
- **파이썬 버전**: 3.10+ 가정, 표준 라이브러리만 사용 (외부 패키지 추가 금지)
- **파일 인터페이스**: 모든 에이전트는 JSON 입출력. 한글 키 사용 가능
- **테스트 코드**: 작성하지 않는다 (책 실습 외 작업 금지)
- **README/CHANGELOG**: 만들지 않는다

---

## 4. 절대 하지 말 것 (Do NOT)

- 새로운 파이썬 패키지를 `pip install` 하지 말 것 (표준 라이브러리로 해결)
- 가상환경(venv)을 만들지 말 것
- Git 저장소 초기화(`git init`) 하지 말 것
- 폴더 구조를 임의로 리팩터링하지 말 것
- `agents/`, `workflows/` 외 새 디렉터리 만들지 말 것
- 책에 없는 추가 기능(날씨 API, 항공권 비교 등) 임의 추가 금지
- 사용자에게 "이 외에 어떤 기능을 원하세요?" 류의 후속 제안을 하지 말 것
- 본 폴더 외부의 파일을 읽거나 쓰지 말 것

---

## 5. 작업 흐름 (사용자 프롬프트 → 행동)

사용자가 각 Step의 프롬프트를 보내면 다음 매핑대로 즉시 수행한다.

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "가족 여행 플래너를 만들 거야" | `travel_input.py` 생성 후 샘플 실행으로 `travel_input.json` 생성 |
| "여행 코스 에이전트", "travel-course-agent" | `agents/travel-course-agent.md` 작성, 웹 검색으로 코스 생성 후 `travel_course.json` 저장 |
| "숙소 추천 에이전트", "accommodation-agent" | `agents/accommodation-agent.md` 작성, 웹 검색으로 숙소 검색 후 `accommodation.json` 저장 |
| "예산 검증" | 세 JSON을 종합해 `budget_summary.json` 생성, 초과 시 §3의 조정 순서 자동 적용 |
| "여행 계획서" | `travel_plan.md` 생성 |
| "워크플로우로 묶어줘" | `workflows/family-travel-workflow.md` 작성 |
| "family-travel-workflow를 실행해줘" | 전체 파이프라인을 순차 실행, 중간 확인 질문 없이 끝까지 진행 |

---

## 6. 실패·모호함 처리 (질문 대신 자동 결정)

| 상황 | 대응 |
|------|------|
| 웹 검색 실패 / 결과 없음 | 한 번만 재시도. 그래도 실패 시 책의 예시 데이터로 채우고 진행 (출력 끝에 "웹 검색 실패로 샘플 데이터 사용"이라고 표시) |
| 예산 초과 | §3의 자동 조정 순서를 적용해 예산 내로 맞춘다. 사용자에게 묻지 않는다 |
| 입력값 누락 | 다음 기본값 사용 — 여행지 "제주도", 기간 "2박 3일", 예산 80만원, 가족 "성인 2명·아이 2명(5세, 8세)", 스타일 "자연 + 아이 체험 위주" |
| 파일이 이미 존재 | 덮어쓴다 (사용자에게 확인하지 않는다) |
| 일반적인 권한 프롬프트 | `.claude/settings.json`에서 이미 허용됨. 다시 묻지 않는다 |
