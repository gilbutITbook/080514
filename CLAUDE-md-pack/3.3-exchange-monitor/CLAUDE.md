# 실습 3.3 — 환율 모니터링 & 환전 타이밍 알림 (exchange-monitor)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙

- **확인 질문을 하지 말 것.**
- **권한은 이미 허용된 것으로 간주한다.**
- **막히면 합리적 기본값으로 계속 진행한다.**
- **즉시 실행한다.**
- **본 폴더 외부는 절대 건드리지 않는다.**
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.

예외적 사용자 확인: 폴더 외부 삭제·이동만.

---

## 1. 프로젝트 목표

USD/JPY/EUR 원화 환율을 매일 자동 수집하고, 목표 환율 도달 또는 전일 대비 급변 시 텔레그램으로 즉시 알림.

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `exchange_config.json` | 모니터링 통화, 목표 환율 범위, 알림 조건, 텔레그램 토큰·chat_id |
| `.env` | 비밀 값 |
| `agents/exchange-collector.md` | 환율 수집 에이전트 (open.er-api.com) |
| `exchange_history.csv` | 누적 환율 이력 (date, currency, rate) |
| `agents/exchange-alerter.md` | 알림 발송 에이전트 |
| `workflows/exchange-workflow.md` | 통합 워크플로우 |
| `logs/exchange_YYYY-MM-DD.md` | 일별 수집·알림 로그 |
| `run_schedule.bat` | 일일 실행용 배치 |

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **수집 API**: open.er-api.com (무료, API 키 없음)
- **모니터링 통화**: USD, JPY, EUR (책 기본)
- **수집 주기**: 1일 1회 (API가 일 1회 갱신)
- **알림 조건 기본값**:
  - USD ≤ 1300원 또는 ≥ 1450원
  - JPY ≤ 900원/100엔 또는 ≥ 1050원/100엔
  - EUR ≤ 1400원 또는 ≥ 1550원
  - 또는 전일 대비 ±1.0% 이상 변동
- **중복 알림 방지**: 동일 통화에 대해 24시간 내 1회만
- **메시지 포맷**: 이모지 사용 💵💴💶, 변화율 포함
- **실행 시각**: 매일 오전 9:00
- **로그 보관**: 최근 90일

---

## 4. 절대 하지 말 것

- 유료 API 키 자동 발급 시도 금지
- 외부 패키지 임의 설치 금지 (`urllib`로 충분)
- 폴더 외부 파일 접근 금지
- 실제 환전 주문 시도 금지
- 책에 없는 통화·기능(가상자산 환율 등) 추가 금지

---

## 5. 작업 흐름 매핑

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "설정 파일 / exchange_config.json" | `exchange_config.json` + `.env` 템플릿 생성 |
| "환율 수집 / exchange-collector" | 에이전트 정의 + 1회 수집으로 `exchange_history.csv` 업데이트 |
| "환율 알림 / exchange-alerter" | 에이전트 정의 + 알림 조건 만족 시 텔레그램 발송 |
| "스케줄 등록" | `run_schedule.bat` + schtasks |
| "워크플로우" | `workflows/exchange-workflow.md` |

---

## 6. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| API 응답 실패 | 1회 재시도, 실패 시 해당일 수집 스킵 |
| 알림 조건 미충족 | 메시지 없이 로그에만 "조건 미충족" 기록 |
| 텔레그램 발송 실패 | 1회 재시도 |
| 통화 누락 | 책 기본값 3종 사용 |
