# 실습 3.1 — 쇼핑 가격 추적 및 최저가 알림 (price-tracker)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙

- **확인 질문을 하지 말 것.**
- **권한은 이미 허용된 것으로 간주한다.**
- **막히면 합리적 기본값으로 계속 진행한다.**
- **즉시 실행한다.**
- **본 폴더 외부는 절대 건드리지 않는다.**
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.

예외적 사용자 확인: 폴더 외부 삭제·이동만.

---

## 1. 프로젝트 목표

`watchlist.json`에 등록한 상품 URL의 가격을 주기적으로 수집·기록하고, 목표가 도달 시 텔레그램 알림과 가격 추이 차트(HTML)를 생성한다.

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `watchlist.json` | 추적 상품 목록 (URL, 상품명, 목표가) |
| `.env` | 텔레그램 토큰·chat_id |
| `agents/price-collector.md` | 가격 수집 에이전트 정의 |
| `agents/price-recorder.md` | 이력 기록 에이전트 정의 |
| `prices.csv` | 누적 가격 이력 (date, product, price) |
| `agents/price-alerter.md` | 목표가 도달 알림 에이전트 정의 |
| `agents/chart-builder.md` | 추이 차트 생성 에이전트 정의 |
| `charts/price_trend_YYYY-MM-DD.html` | 일별 추이 차트 |
| `workflows/price-workflow.md` | 통합 워크플로우 정의 |
| `run_schedule.bat` | 주기 실행용 배치 |

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **인코딩**: UTF-8
- **가격 단위**: 원(₩), 정수
- **수집 주기**: 1시간마다 (스케줄러 등록 시)
- **차트 라이브러리**: Chart.js CDN (외부 패키지 설치 불필요)
- **알림 조건**: 현재가 ≤ 목표가
- **중복 알림 방지**: 같은 상품에 대해 24시간 내 1회만 알림
- **알림 메시지 포함**: 상품명, 현재가, 목표가, 구매 링크
- **수집 실패한 상품**: 해당 회차만 스킵, CSV에는 빈 행을 남기지 않는다
- **차트 보관**: 최근 30일

---

## 4. 절대 하지 말 것

- 자동 구매·결제 시도 절대 금지
- 사이트 robots.txt 무시 금지 (수집 차단 시 즉시 중단)
- 외부 패키지 임의 설치 금지 (오픈클로 + 표준 라이브러리)
- 폴더 외부 파일 접근 금지
- 책에 없는 기능(가격 예측 모델 등) 추가 금지

---

## 5. 작업 흐름 매핑

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "watchlist.json / 추적 상품 목록" | `watchlist.json` 생성 (책 예시 상품) |
| "가격 수집 / price-collector" | 에이전트 정의 + 1회 수집 실행 |
| "이력 기록 / price-recorder" | 에이전트 정의 + `prices.csv` 생성 |
| "목표가 알림 / price-alerter" | 에이전트 정의 + 텔레그램 발송 |
| "차트 / chart-builder" | 에이전트 정의 + `charts/` HTML 생성 |
| "워크플로우" | `workflows/price-workflow.md` |
| "스케줄 등록" | `run_schedule.bat` + schtasks |

---

## 6. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| 가격 파싱 실패 | 1회 재시도, 그래도 실패 시 해당 회차 스킵하고 로그에 기록 |
| 접속 차단(403/429) | 즉시 중단, 다음 회차로 미룸 |
| 텔레그램 발송 실패 | 1회 재시도, 실패 시 로그만 기록 |
| `watchlist.json` 비어 있음 | 책 예시 상품 1개로 자동 채움 |
