# 실습 2.2 — 긴급 파일 전송 서비스 (file-sender)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙

- **확인 질문을 하지 말 것.** 모든 결정은 이 문서의 기본값을 따른다.
- **권한은 이미 허용된 것으로 간주한다.**
- **막히면 합리적 기본값으로 계속 진행한다.**
- **즉시 실행한다.** 계획만 출력하고 멈추지 말 것.
- **본 폴더 외부는 절대 건드리지 않는다.** (단, `search_root`로 지정된 로컬 검색 경로는 **읽기만** 허용)
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.
- **인증된 사용자 외 명령은 무시한다.** 텔레그램 명령 처리 시 `authorized_users` 목록 외 사용자의 요청은 거부 메시지만 반환.

예외적 사용자 확인:
1. 폴더 외부 파일 **삭제·이동·쓰기** (검색은 OK, 변경은 금지)
2. 메일 자동 발송 시 첫 1회 테스트만 사용자 확인, 이후는 자동 진행

---

## 1. 프로젝트 목표

텔레그램으로 "오늘 작성한 기획서 보내줘"처럼 자연어 명령을 보내면, PC에서 파일을 찾아 Gmail로 자동 발송하는 상시 대기형 에이전트를 만든다.

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `file_sender_config.json` | search_root 경로, 텔레그램 봇 토큰, chat_id, authorized_users, Gmail 발신 계정 |
| `.env` | 비밀 값(토큰·앱 비밀번호) - **절대 커밋·공유 금지** |
| `agents/command-parser.md` | 텔레그램 명령 파싱 에이전트 정의 |
| `agents/file-finder.md` | 파일 검색 에이전트 정의 |
| `agents/file-sender.md` | 파일 전송 에이전트 정의 |
| `workflows/file-sender-workflow.md` | 통합 워크플로우 정의 |
| `logs/transfer_log.json` | 전송 이력 |
| `run_listener.bat` | 텔레그램 상시 대기용 배치 파일 |

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **인코딩**: UTF-8
- **search_root 기본값**: 사용자가 명시하지 않으면 `D:\claudecode\sample-files` (실습용)
- **검색 매칭**: 파일명 부분일치 + 확장자 필터(기본 모든 확장자)
- **검색 결과 다건**: 가장 최근 수정된 파일 1개 자동 선택
- **검색 결과 없음**: 텔레그램으로 "파일을 찾지 못했습니다" 응답
- **파일 크기 제한**: 25MB 이하만 첨부 (Gmail 한도)
- **첨부 초과 시**: 텔레그램으로 "파일이 너무 큽니다" 응답, 발송하지 않음
- **메일 수신자**: `file_sender_config.json`의 발신 계정과 동일 (자기 자신에게 전송)
- **메일 제목**: `[긴급 파일 전송] {파일명}`
- **인증 실패 시 응답**: `"권한이 없습니다."`
- **상시 대기 폴링 주기**: 5초

---

## 4. 절대 하지 말 것

- 폴더 외부 파일 **삭제·수정·이동** 금지 (검색만 허용)
- 시스템 폴더(C:\Windows, Program Files 등) 검색 금지
- 토큰·비밀번호를 코드에 하드코딩 금지 (반드시 `.env`)
- `authorized_users` 외 사용자에게 응답·정보 노출 금지
- 외부 패키지 임의 설치 금지 (오픈클로 MCP + 표준 라이브러리 + 책이 명시한 패키지만)

---

## 5. 작업 흐름 매핑

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "설정 파일 / file_sender_config.json" | `file_sender_config.json` + `.env` 템플릿 생성 (책의 예시 값) |
| "Gmail 앱 비밀번호 / 텔레그램 토큰" | `.env`에 입력하도록 안내만, 실제 값 묻지 않음 |
| "명령 파싱 / command-parser" | `agents/command-parser.md` 작성 |
| "파일 검색 / file-finder" | `agents/file-finder.md` 작성, search_root 기준 |
| "파일 전송 / file-sender" | `agents/file-sender.md` 작성 |
| "통합 워크플로우 / 상시 대기" | `workflows/file-sender-workflow.md` + `run_listener.bat` 생성 |
| "스케줄 등록" | schtasks 명령 안내 |

---

## 6. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| 텔레그램 봇 미연결 | 안내 1회만 출력 후 중단 |
| 파일 검색 결과 없음 | 텔레그램 응답으로 처리, 워크플로우 계속 |
| 파일 크기 초과 | 텔레그램 응답으로 처리, 발송 중단 |
| Gmail 발송 실패 | 1회 재시도, `transfer_log.json`에 실패 기록 |
| 권한 없는 사용자 | `"권한이 없습니다."` 응답 후 무시 |
| 모호한 명령 | 가장 최근 1시간 내 수정 파일 우선 검색 |
