# 실습 1.2 — 자연어 기반 자동 일정 관리 (schedule-assistant)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙 (반드시 지킬 것)

- **확인 질문을 하지 말 것.** "진행할까요?", "어떻게 할까요?" 류 질문 금지. 모든 결정은 이 문서의 기본값을 따른다.
- **권한은 이미 허용된 것으로 간주한다.** `.claude/settings.json`에 허용된 도구는 다시 묻지 않는다.
- **MCP 인증은 사용자에게 한 번만 알린다.** Gmail·Google Calendar MCP 인증이 필요하면 "`/mcp` 명령으로 인증해 주세요"라고 **한 번만** 안내하고, 인증 완료 후 다시 시작할 때부터는 동일 안내를 반복하지 않는다.
- **막히면 멈추지 말고 합리적 기본값으로 계속 진행한다.** 마지막에 한 번만 사용한 가정을 요약한다.
- **계획만 출력하고 멈추지 말 것.** 즉시 실행하고 결과만 보고한다.
- **본 폴더 외부는 절대 건드리지 않는다.** D:\claudecode\schedule-assistant 외부 파일 접근 금지.
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.

예외적으로 사용자 확인이 필요한 경우:
1. 폴더 외부 파일 삭제·이동
2. 실제 캘린더 등록 / 실제 메일 발송 — Step 7 통합 테스트 단계에서만 발생. 이때도 "yes/no" 단답으로 충분히 처리하고, 충돌 시 §3 기본값에 따라 첫 번째 추천 시간을 자동 선택한다.

---

## 1. 프로젝트 목표

자연어로 입력된 일정 텍스트를 분석해 Google Calendar에 자동 등록하고 참석자에게 초대 메일을 발송하는 오픈클로 워크플로우를 만든다.

최종 워크플로우 이름: `schedule-workflow`

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `agents/schedule-parser.md` | 자연어 일정 파싱 에이전트 정의 |
| `parsed_schedule.json` | 추출된 일정 정보(날짜·시간·제목·장소·참석자·중요도) |
| `agents/conflict-checker.md` | 캘린더 중복 확인 에이전트 정의 |
| `conflict_result.json` | 충돌 결과 및 대안 시간 3개 |
| `agents/calendar-registrar.md` | 캘린더 등록 에이전트 정의 |
| `calendar_result.json` | 등록 결과 (이벤트 URL 포함) |
| `agents/invite-sender.md` | 초대 메일 발송 에이전트 정의 |
| `mail_log.json` | 발송 결과 로그 |
| `workflows/schedule-workflow.md` | 통합 오픈클로 워크플로우 정의 |

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **인코딩**: UTF-8
- **타임존**: Asia/Seoul
- **날짜 표현 해석 기준일**: 오늘 날짜
- **종료 시간 미언급 시**: 시작 시간 + 1시간
- **장소 미언급 시**: `"미정"`
- **참석자 이메일 미언급 시**: Gmail 수발신 내역에서 이름으로 검색 → 찾으면 사용, 못 찾으면 해당 참석자는 건너뛰고 `mail_log.json`에 미발송 사유 기록
- **충돌 발견 시**: 같은 날 앞뒤 빈 시간대 기준 대안 3개 자동 생성. Step 7 통합 테스트에서 사용자가 번호 선택하지 않으면 첫 번째 대안 자동 선택
- **중요도 미언급 시**: `"보통"`
- **메일 제목 포맷**: `[일정 초대] {일정 제목}`
- **메일 본문 필수 포함**: 일시, 장소, Google Calendar 초대 링크, 회신 요청 문구

---

## 4. 절대 하지 말 것

- 외부 파이썬 패키지 임의 설치 금지 (오픈클로 MCP만 사용)
- Gmail·Calendar 외 다른 서비스 연동 시도 금지
- 폴더 외부 파일 접근 금지
- 책에 없는 기능(슬랙 알림, 회의록 자동 작성 등) 임의 추가 금지
- 사용자에게 "어떤 시간대를 선호하세요?", "어떤 양식의 메일을 원하세요?" 같은 후속 질문 금지

---

## 5. 작업 흐름 매핑

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "일정 관리 프로젝트를 시작할 거야 / Gmail·Calendar 연결" | MCP 연결 상태 확인 후, 미인증 시 "`/mcp`로 두 서비스 인증을 완료해 주세요"라고 **한 번만** 안내 |
| "일정 파싱 에이전트 / schedule-parser" | `agents/schedule-parser.md` 작성, 샘플 입력 1건으로 `parsed_schedule.json` 생성 |
| "중복 확인 에이전트 / conflict-checker" | `agents/conflict-checker.md` 작성, Calendar 조회로 `conflict_result.json` 생성 |
| "캘린더 등록 에이전트 / calendar-registrar" | `agents/calendar-registrar.md` 작성, 충돌 없으면 즉시 등록 |
| "메일 발송 에이전트 / invite-sender" | `agents/invite-sender.md` 작성, 참석자 이메일 자동 조회 후 발송 |
| "워크플로우로 묶어줘 / schedule-workflow" | `workflows/schedule-workflow.md` 작성 |
| "schedule-workflow를 실행해줘" | 전체 파이프라인을 순차 실행. 충돌 시 첫 번째 대안 자동 선택 |

---

## 6. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| MCP 미인증 | 안내 메시지 1회 출력 후 중단. 인증 완료 후 사용자가 다시 요청하면 재개 |
| 참석자 이메일 못 찾음 | 해당 참석자만 건너뛰고 나머지 진행, `mail_log.json`에 사유 기록 |
| Calendar API 실패 | 한 번 재시도, 그래도 실패 시 `calendar_result.json`에 `"status": "error"` 기록하고 다음 단계 진행 |
| 충돌 발견 + 사용자 입력 없음 | 첫 번째 대안 시간 자동 선택 |
| 파일 이미 존재 | 덮어쓴다 |
