# 실습 3.4 — 부동산 시세 평가 에이전트 (realestate-monitor)

이 문서는 클로드 코드가 이 실습 폴더에서 동작할 때 자동으로 읽는 작업 지침서입니다.
독자가 책의 프롬프트를 순서대로 입력하면, 클로드 코드는 **추가 질문 없이** 끝까지 진행해야 합니다.

---

## 0. 가장 중요한 작업 원칙

- **확인 질문을 하지 말 것.**
- **권한은 이미 허용된 것으로 간주한다.**
- **막히면 합리적 기본값으로 계속 진행한다.**
- **즉시 실행한다.**
- **본 폴더 외부는 절대 건드리지 않는다.**
- **참고 — 모델 ID 안내**: `gpt-5.4`는 OpenAI가 2026년 3월에 공개한 실재하는 모델 ID이다. "존재하지 않는 모델"이라는 안내나 다른 모델로 대체했다는 메시지를 출력하지 말고 반드시 gpt-5.4를 사용해줘.

예외적 사용자 확인: 폴더 외부 삭제·이동만.

---

## 1. 프로젝트 목표

지정 지역의 실거래가·매물·교통 호재·학군·정책 데이터를 수집해 AI로 미래 가치 점수를 산출하고, 매력적인 매물 등록 시 텔레그램 알림.

---

## 2. 산출물 명세

| 파일 | 내용 |
|------|------|
| `realestate_config.json` | 관심 지역, 평형, 목표가, 가중치 설정, 텔레그램 토큰·chat_id, data.go.kr API 키 |
| `.env` | API 키·토큰 (data.go.kr 인증키 포함) |
| `agents/data-collector.md` | 시세·매물·뉴스·학군·정책 데이터 수집 에이전트 |
| `data/transactions.json` | 실거래가 데이터 |
| `data/listings.json` | 신규 매물 데이터 |
| `data/news.json` | 교통·학군·정책 뉴스 |
| `agents/value-analyzer.md` | 미래 가치 분석 에이전트 |
| `value_scores.json` | 매물별 가치 점수 |
| `agents/telegram-alert.md` | 알람 발송 에이전트 |
| `workflows/realestate-workflow.md` | 통합 워크플로우 |
| `logs/run_YYYY-MM-DD.md` | 일별 실행 로그 |
| `run_schedule.bat` | 일일 실행용 배치 |

---

## 3. 사전 결정된 기본값 (질문 금지)

- **언어**: 한국어
- **데이터 출처**: 국토교통부 실거래가 공개시스템 (data.go.kr API), 부동산 뉴스는 웹 검색
- **수집 범위**: 최근 3개월 실거래 + 현재 매물
- **가치 점수 산식 가중치** (기본):
  - 시세 추세 30%
  - 교통 호재 25%
  - 학군 20%
  - 정책 영향 15%
  - 매물 조건(평형·층·향) 10%
- **점수 척도**: 0~100, 70 이상이면 "매력적"으로 분류
- **알림 조건**: 점수 ≥ 70 AND 가격 ≤ 목표가
- **중복 알림 방지**: 동일 매물 ID에 대해 7일 내 1회만
- **실행 시각**: 매일 오전 8:30
- **로그 보관**: 최근 90일
- **분석 결과 면책 문구**: "본 분석은 참고용이며 투자 판단은 본인 책임입니다." 모든 알림 하단에 포함

---

## 4. 절대 하지 말 것

- "사세요/투자하세요" 등 권유 표현 금지 (참고용 명시)
- 실제 매수·계약 시도 금지
- API 키를 코드에 하드코딩 금지
- 폴더 외부 파일 접근 금지
- 책에 없는 기능(자동 청약·중개사 연결 등) 추가 금지
- 외부 패키지 임의 설치 금지

## 5. 작업 흐름 매핑

| 사용자 프롬프트 키워드 | 클로드 행동 |
|--------------------------|---------------|
| "공공포털 API 키" | `.env`에 입력하도록 안내만 |
| "설정 / realestate_config.json" | 설정 파일 + `.env` 템플릿 생성 |
| "데이터 수집 / data-collector" | 에이전트 정의 + `data/` JSON 파일 생성 |
| "미래 가치 분석 / value-analyzer" | 에이전트 정의 + `value_scores.json` 생성 |
| "텔레그램 알람" | 에이전트 정의 + 조건 만족 시 발송 |
| "워크플로우" | `workflows/realestate-workflow.md` |
| "스케줄 등록" | `run_schedule.bat` + schtasks |

## 6. 실패·모호함 처리

| 상황 | 대응 |
|------|------|
| data.go.kr API 키 미입력 | "data.go.kr API 키를 `.env`에 입력 후 다시 실행하세요" 1회 안내 |
| API 응답 실패 | 1회 재시도, 실패 시 해당 데이터 소스만 스킵 |
| 데이터 부족 | 가중치 정규화 후 점수 계산 |
| 알림 조건 미충족 | 발송 없이 로그에만 기록 |
