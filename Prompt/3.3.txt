exchange-monitor 폴더에 rate-collector 에이전트를 만들어 줘.
이 에이전트는:
[환율 수집]
- https://open.er-api.com/v6/latest/USD에 GET 요청을 보낸다.
- 응답 JSON에서 rates.KRW, rates.JPY, rates.EUR 값을 추출한다.
- JPY는 100엔 기준으로 변환한다.
계산식: JPY100_KRW = rates.KRW / rates.JPY * 100
- 수집 시각(KST)과 함께 결과를 딕셔너리로 구성한다.
형식: {"date": "2026-04-14", "USD": 1350.5, "JPY100": 912.3, "EUR": 1510.2}
[이력 저장]
- exchange_config.json의 data_file(exchange_history.json)을 읽는다.
- 파일이 없으면 빈 리스트 []로 초기화한다.
- 오늘 날짜 데이터가 이미 있으면 덮어쓴다. (중복 방지)
- 최근 30일 데이터만 유지하고 오래된 것은 삭제한다.
- 업데이트된 리스트를 exchange_history.json에 저장한다.



exchange-monitor 폴더에 rate-alert 에이전트를 만들어 줘.
이 에이전트는:
[데이터 로드]
- exchange_history.json에서 오늘과 어제 데이터를 읽는다.
- .env에서 TELEGRAM_BOT_TOKEN, TELEGRAM_CHAT_ID를 로드한다.
- exchange_config.json에서 targets 목록과 change_alert_pct를 읽는다.
[조건 분석]
- targets 각 통화에 대해 두 가지 조건을 검사한다:
조건 1 목표가 도달: 현재가가 alert_below 이하 또는 alert_above 이상
조건 2 급변 감지: 전일 대비 변동률이 change_alert_pct 이상
- 어제 데이터가 없으면 조건 2는 건너뛴다.
[텔레그램 발송]
- 알림 대상이 하나라도 있으면 단일 메시지를 구성한다:
[환율 알림] {오늘날짜}
(조건 1 충족 통화마다)
{display} 목표가 도달!
현재: {현재가}원 (목표 기준: {alert_below 또는 alert_above}원)
전일 대비: {방향} {변동액}원 ({변동률}%)
→ {tip_below 또는 tip_above}
(조건 2만 충족 통화마다)
{display} 급변 감지! 현재: {현재가}원 전일 대비: {방향} {변동액}원 ({변동률}%)
오늘 전체 환율
• USD/KRW: {USD}원
• JPY/KRW: {JPY100}원 (100엔 기준)
• EUR/KRW: {EUR}원
- 알림 대상이 없으면 텔레그램을 발송하지 않는다.
- 모든 메시지는 반드시 한국어로 작성한다.



지금까지 만든 에이전트를 하나의 워크플로로 묶어 줘.
워크플로 이름: exchange-workflow
[중요] 이 워크플로는 클로드 코드의 슬래시 명령어
(/exchange-workflow)로 호출되므로, 반드시 다음 경로에 생성해 줘:
.claude/commands/exchange-workflow.md
(.claude/commands/ 폴더가 없으면 함께 생성)
실행 순서 (각 단계는 직전 산출물에 의존하므로 순차 실행):
1. rate-collector 실행 → exchange_rates.json 저장
2. rate-alert 실행 → 목표 환율 도달 여부 확인 및 텔레그램 알림 (누락 금지)
- 실제 호출 명령: python send_telegram.py
(또는 rate-alert가 사용하는 실제 스크립트명)
- 정의 파일(agents/rate-alert.md)을 읽기만 하지 말고,
반드시 실제 스크립트를 호출해 텔레그램 발송까지 완료한다.
- 발송 결과(성공/실패, 메시지 길이)를 콘솔에 출력한다.
- 실패 시 1회 재시도 → logs/telegram_YYYY-MM-DD.log 기록 후
다음 단계 진행 (전체 중단 금지)
3. 완료 메시지 출력: '환율 확인 완료 - {날짜/시간}'
수동 실행 트리거 (둘 다 동일 동작):
- 슬래시 명령어: /exchange-workflow
- 자연어: '지금 바로 환율 수집하고 알림 조건 확인해서 텔레그램 발송해 줘'
[필수] 위 워크플로 파일을 만든 직후, 사용자에게 묻지 말고
즉시 /exchange-workflow 를 한 번 실행해서 전체 파이프라인이
정상 동작하는지 검증하고 텔레그램 발송까지 완료해 줘.
"실행할까요?", "진행해도 될까요?" 같은 확인 질문 절대 금지.
만들고 → 바로 실행 → 결과 보고.



exchange-monitor 폴더에 run.bat 파일을 만들고,
Windows 작업 스케줄러에 ExchangeMonitor-Daily 이름으로 등록해 줘.
run.bat 파일 내용 (아래 코드 그대로 작성):
@echo off
setlocal
set "BASE_DIR=D:\claudecode\exchange-monitor"
set "LOG_DIR=%BASE_DIR%\logs"
set "LOG_FILE=%LOG_DIR%\run.log"
if not exist "%LOG_DIR%" mkdir "%LOG_DIR%"
echo. >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
echo [%date% %time%] ExchangeMonitor-Daily start >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
set "ANTHROPIC_API_KEY="
cd /d "%BASE_DIR%"
call claude -p "지금 바로 환율 수집하고 알림 조건 확인해서 텔레그램 발송해 줘." --permission-mode
bypassPermissions >> "%LOG_FILE%" 2>&1
echo [%date% %time%] ExchangeMonitor-Daily end (exit: %errorlevel%) >> "%LOG_FILE%"
endlocal
schtasks /create /tn "ExchangeMonitor-Daily" /tr "cmd /c D:\claudecode\exchange-monitor\run.
bat" /sc weekly /d mon,tue,wed,thu,fri /st 17:20 /f
등록 후 schtasks /query /tn \"ExchangeMonitor-Daily\"으로 확인해 줘.
run.bat 파일은 반드시 CRLF(Windows) 줄 끝 형식으로 저장해 줘.
