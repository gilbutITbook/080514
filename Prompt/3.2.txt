오늘의 핫이슈 뉴스를 수집하는 에이전트를 만들어 줘. 이름은 news-collector.
수집 대상 (해외 우선):
- 해외 (우선): Reuters, Bloomberg, CNBC, Financial Times, WSJ
- 국내 (보조): 연합뉴스, 한국경제 (해외 이슈의 국내 영향 파악용)
수집 기준 (주식에 영향을 주는 뉴스 우선):
- 지정학적 이슈 (전쟁, 제재, 외교 분쟁)
- 경제 정책 (금리, 환율, 무역 협정)
- 산업 이슈 (반도체, 2차전지, 바이오, AI)
- 기업 이슈 (실적 발표, 합병, 규제)
수집 방법: 웹 검색 도구 사용
수집 건수: 오늘 날짜 기준 TOP 5 핫이슈
각 뉴스별 저장 형식:
- 제목, 출처, 날짜
- 핵심 내용 2~3줄 요약
- 관련 키워드 (예: 지정학, 반도체, 금리)
결과를 today_news.json으로 저장해 줘.



주식 영향 분석 에이전트를 만들어 줘. 이름은 impact-analyzer.
today_news.json의 각 뉴스를 읽고 아래 형식으로 분석해 줘.
각 뉴스별 분석 항목:
1. 강세 예상 종목/섹터:
- 이 뉴스가 왜 해당 종목에 유리한지 (1~2줄)
- 대표 종목 3개 (국내주 우선, 해외주 포함 가능)
2. 약세 예상 종목/섹터:
- 이 뉴스가 왜 해당 종목에 불리한지 (1~2줄)
- 대표 종목 3개 (국내주 우선, 해외주 포함 가능)
3. 영향 강도: 높음 / 보통 / 낮음
- 높음: 직접적 가격 영향 예상
- 보통: 간접적 영향, 시장 반응 관찰 필요
- 낮음: 장기적 영향, 즉각 반응 미미
분석 시 주의사항:
- 추측이 아닌 경제적 인과 관계에 근거할 것
- 투자 권유가 아닌 참고용임을 명시할 것
- 국내·해외 시장 모두 고려할 것
결과를 impact_analysis.json으로 저장해 줘.



오늘의 투자 흐름 리포트 에이전트를 만들어 줘. 이름은 report-printer.
today_news.json과 impact_analysis.json을 읽어서
아래 형식으로 출력하고 daily_report.md에 저장해 줘.
출력 형식:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
오늘의 뉴스 기반 주식 영향 분석
날짜: YYYY-MM-DD
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[이슈 1] 뉴스 제목 (출처)
강세 예상: 종목A, 종목B, 종목C
→ 이유: 한 줄 설명
약세 예상: 종목D, 종목E, 종목F
→ 이유: 한 줄 설명
영향 강도: 높음
[이슈 2] ...
─────────────────────────────
오늘의 한 줄 요약:
(전체 뉴스를 종합한 오늘 시장 흐름 요약)
─────────────────────────────
본 분석은 참고용이며 투자 권유가 아닙니다.
결과를 터미널에 출력하고 daily_report.md로도 저장해 줘.



텔레그램 발송 에이전트를 만들어 줘. 이름은 telegram-sender.
daily_report.md 내용을 텔레그램 봇으로 발송해 줘.
메시지 형식:
오늘의 뉴스 기반 주식 영향 분석
날짜: YYYY-MM-DD
─────────────────────────
[이슈 1] 뉴스 제목
강세: 종목A, 종목B, 종목C
약세: 종목D, 종목E, 종목F
영향 강도: 높음
...(이슈 2~5 동일 형식)
─────────────────────────
오늘의 한 줄 요약: (요약 내용)
본 분석은 참고용이며 투자 권유가 아닙니다.
설정값은 .env 파일에 저장해 줘:
TELEGRAM_BOT_TOKEN=여기에_봇_토큰_입력
TELEGRAM_CHAT_ID=여기에_채팅_ID_입력
파이썬 requests 라이브러리를 사용해 줘.
채팅 ID는 봇에 메시지를 보낸 후
https://api.telegram.org/bot{TOKEN}/getUpdates에서 확인 가능.



지금까지 만든 에이전트(news-collector, impact-analyzer, report-printer,
telegram-sender)를 순서대로 실행하는 슬래시 명령을 만들어 줘.
[중요] 이 워크플로는 클로드 코드의 슬래시 명령어
(/news-stock-workflow)로 호출되므로, 반드시 다음 경로에 생성해 줘:
.claude/commands/news-stock-workflow.md
(.claude/commands/ 폴더가 없으면 함께 생성)
실행 순서 (각 단계는 직전 단계 산출물에 의존하므로 반드시 순차 실행):
1. news-collector 실행 → today_news.json 생성
(실패 시 기존 today_news.json이 있으면 그것으로 다음 단계 진행)
2. impact-analyzer 실행 → impact_analysis.json 생성
3. report-printer 실행 → daily_report.md 저장 + 콘솔 출력
4. telegram-sender 실행 → 텔레그램 발송
- 실제 호출 명령: python send_telegram.py
- 정의 파일(agents/telegram-sender.md)을 읽기만 하지 말고,
반드시 python send_telegram.py를 호출해 실제 발송까지 완료한다.
- 발송 결과(성공/실패, 메시지 길이)를 콘솔에 출력한다.
- 실패 시 1회 재시도 → logs/telegram_YYYY-MM-DD.log 기록 후
다음 단계 진행 (전체 중단 금지).
5. 완료 메시지 출력 + daily_report.md 자동 오픈
(PowerShell: Start-Process .\daily_report.md)
수동 실행 트리거 (둘 다 동일 동작):
- 슬래시 명령어: /news-stock-workflow
- 자연어: '지금 뉴스 분석해 줘'
[필수] 위 워크플로 파일을 만든 직후, 사용자에게 묻지 말고
즉시 /news-stock-workflow 를 한 번 실행해서 전체 파이프라인이
정상 동작하는지 검증하고 텔레그램 발송까지 완료해 줘.
"실행할까요?" "진행해도 될까요?" 같은 확인 질문 절대 금지.
만들고 → 바로 실행 → 결과 보고.



news-stock-analyzer 폴더에 run.bat 파일을 만들고,
Windows 작업 스케줄러에 NewsStockAnalyzer-Evening 이름으로 등록해 줘.
run.bat 파일 내용 (아래 코드 그대로 작성):
@echo off
setlocal
set "BASE_DIR=D:\claudecode\news-stock-analyzer"
set "LOG_DIR=%BASE_DIR%\logs"
set "LOG_FILE=%LOG_DIR%\run.log"
if not exist "%LOG_DIR%" mkdir "%LOG_DIR%"
echo. >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
echo [%date% %time%] NewsStockAnalyzer-Evening start >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
set "ANTHROPIC_API_KEY="
cd /d "%BASE_DIR%"
call claude -p "/news-stock-workflow" --permission-mode bypassPermissions >> "%LOG_FILE%" 2>&1
echo [%date% %time%] NewsStockAnalyzer-Evening end (exit: %errorlevel%) >> "%LOG_FILE%"
endlocal
schtasks /create /tn "NewsStockAnalyzer-Evening" /tr "cmd /c D:\claudecode\news-stock-analyzer\
run.bat" /sc weekly /d mon,tue,wed,thu,fri /st 17:00 /f
등록 후 schtasks /query /tn \"NewsStockAnalyzer-Evening\"으로 확인해 줘.
run.bat 파일은 반드시 CRLF(Windows) 줄 끝 형식으로 저장해 줘.
