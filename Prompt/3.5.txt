sns-trend-monitor 폴더에 trend_config.json과 .env 파일을 생성해 줘.
trend_config.json 내용:
- "topics": ["AI", "LLM", "개발자 도구", "테크"] (모니터링할 분야 목록)
- linkedin_post_count: 30 (분야별 수집할 LinkedIn 게시물 수)
- linkedin_language: "ko" (LinkedIn 검색 언어, ko=한국어 우선)
- x_result_count: 50 (X 트렌딩 수집 건수)
- top_keywords_count: 20 (분야별 상위 키워드 수)
- trending_rise_threshold: 3 (급상승 기준: 순위 3단계 이상 상승)
- collect_interval_hours: 24 (24시간마다 수집)
- report_day: "monday"
- report_time: "08:00"
- subscribers: ["user@gmail.com"]
- telegram_alert: true
검색은 오픈클로의 웹 검색 도구를 사용해 줘.
.env 내용:
LINKEDIN_COOKIE=여기에_LinkedIn_쿠키_입력
TELEGRAM_BOT_TOKEN=여기에_텔레그램_봇_토큰_입력
TELEGRAM_CHAT_ID=여기에_채팅_ID_입력
GMAIL_SENDER=여기에_Gmail_이메일_입력
GMAIL_APP_PASSWORD=여기에_Gmail_앱_비밀번호_16자리_입력
OPENAI_API_KEY=여기에_OpenAI_키_입력
TAVILY_API_KEY=여기에_Tavily_키_입력
.gitignore도 생성해서 .env를 추가해 줘.



sns-trend-monitor 폴더에 keyword-collector 에이전트를 만들어 줘.
이 에이전트는:
[1단계: 원문 수집]
- Tavily API 키는 환경 변수 TAVILY_API_KEY 참조
- X(트위터): Tavily API로 topics별 최신 게시물 수집
쿼리: '{토픽} new AI tool release OR launch OR update 2026 site:x.com'
실패 시 Nitter RSS로 대체 수집
URL: https://nitter.privacydev.net/search/rss?q={토픽키워드}&f=tweets
- LinkedIn: Tavily API로 topics별 최신 게시물 수집
쿼리: site:linkedin.com/posts "{토픽}" new release OR launch OR update 2026
[2단계: OpenAI API로 기술 키워드 추출]
- 수집한 원문을 아래 파이썬 코드로 OpenAI API에 보내 기술 키워드만 추출한다
import openai, os, json
client = openai.OpenAI(api_key=os.environ['OPENAI_API_KEY'])
extract_prompt = f'''아래 SNS 게시물에서 언급된
AI 제품명, 모델명, 프레임워크명, 서비스명만 추출해 JSON으로 반환해.
플랫폼명(twitter, linkedin, x), 일반어(ai, 최신, 블로그, 실시간)는 제외.
한국어 조사(~의 ~를 ~에서)가 붙으면 제거 후 판단.
출력 형식: {"keywords": ["Claude 4", "MCP", "Grok 3", ...]}
게시물 목록:
{posts_text}'''
resp = client.chat.completions.create(
model='gpt-5.4-mini',
response_format={'type': 'json_object'},
messages=[{'role': 'user', 'content': extract_prompt}]
)
extracted = json.loads(resp.choices[0].message.content)['keywords']
- topics별로 위 코드를 실행하고 키워드 등장 횟수를 합산한다.
- 1개 topic에서만 1회 등장한 키워드는 제외한다.
[3단계: 결과 저장]
- 추출된 기술 용어를 언급 횟수 기준 내림차순으로 정렬한다.
- 결과를 raw_keywords_{날짜}.json 으로 저장한다.
형식: { "topic": "테크", "source": "x_linkedin",
"keywords": [{"word": "Claude 4", "count": 12}, ...] }



sns-trend-monitor 폴더에 trend-analyzer 에이전트를 만들어 줘.
이 에이전트는:
[1단계: 데이터 읽기]
- trend_config.json에서 trending_rise_threshold, top_keywords_count 값을 읽는다.
- raw_keywords_{오늘날짜}.json (이번 주 데이터)를 읽는다.
- raw_keywords_{7일전날짜}.json이 있으면 읽고, 없으면 ‘지난 주 없음’으로 처리한다.
(첫 실행 시: 이번 주 데이터만으로 분석 진행, 순위 변화 없이 전체 신규 처리)
[2단계: 키워드 분류]
- 분야별로 아래 4가지를 분류한다
· 급상승: 순위가 trending_rise_threshold 이상 올라간 키워드
· 상승: 순위가 1~2단계 올라간 키워드
· 하락: 순위가 내려간 키워드
· 신규: 지난 주 TOP 20에 없었으나 이번 주 진입한 키워드
- 지난 주 데이터가 없으면 모든 키워드를 신규로 처리한다.
[3단계: 트렌드 요약]
- 전체 트렌드 흐름을 2~3문장으로 요약한다.
예: '이번 주 테크 분야에서는 AI 에이전트 관련 키워드가 급부상했으며...'
[4단계: 결과 저장]
- 모든 파일 입출력 시 encoding='utf-8' 명시한다.
- 결과를 trend_report_{오늘날짜}.json으로 저장한다.
형식:
{
"date": "2026-04-27",
"topics": [{"topic": "AI 테크",
"rising": [{"word": "Claude 4", "rank_change": 5}],
"up": [{"word": "MCP", "rank_change": 2}],
"down": [{"word": "GPT-4", "rank_change": -3}],
"new": [{"word": "Grok 3"}],
"summary": "이번 주 테크 분야에서는..."}]
}



sns-trend-monitor 폴더에 report-composer 에이전트를 만들어 줘.
이 에이전트는 trend_report_{오늘날짜}.json을 읽어
HTML 보고서(trend_report.html)와 텔레그램 메시지(telegram_summary.txt)를 생성한다.
입력 파일 스키마 (trend-analyzer가 생성한 형식):
{
"date": "2026-04-27",
"topics": [{"topic": "AI 테크",
"rising": [{"word": "Claude 4", "rank_change": 5}],
"up": [...], "down": [...],
"new": [{"word": "Grok 3"}],
"summary": "이번 주 테크 분야에서는..."}]
}
=== HTML 이메일 보고서 구조 ===
[헤더]
배경 #1A1A2E, 흰색 텍스트
제목: SNS 트렌드 위클리 — {날짜} 기준
부제: {분석 대상 분야} | X · LinkedIn
[섹션 1] 이번 주 급상승 키워드 TOP 5
분야별로 가로 카드 나열 (배경 #FF6B6B→#FFF5F5 그라데이션)
카드마다: 순위 뱃지, 키워드, 전주 대비 상승폭(▲N), 출처 아이콘(X/LinkedIn)
[섹션 2] 분야별 트렌드 변화표
분야마다 테이블: 키워드 | 이번 주 빈도 | 전주 빈도 | 변화 | 분류
급상승은 행 배경 #FFF3E0, 신규는 행 배경 #E8F5E9
[섹션 3] AI 트렌드 해석
각 분야별 2~3문장 요약 (trend-analyzer가 생성한 흐름 설명 사용)
배경 #F8F9FA, 왼쪽 테두리 #4A90D9
[푸터] 수집 소스 명시, 자동 생성 안내
=== 텔레그램 요약 (단일 메시지) ===
[SNS 트렌드 요약] {날짜}
─────────────────────────
분야별로:
{분야 이모지} {분야명}
급상승: {키워드1}(▲N위) · {키워드2}(▲N위)
신규진입: {키워드}
{2줄 트렌드 요약}
맨 아래: 전체 보고서는 이메일로 발송됩니다.



sns-trend-monitor 폴더에 notifier 에이전트를 만들어 줘.
이 에이전트는:
- trend_report.html과 telegram_summary.txt를 읽는다.
- Gmail로 이메일 발송:
· 수신자: trend_config.json의 subscribers 전원
· 제목: SNS 트렌드 위클리 | {날짜} — {분야} 급상승 키워드
· 본문: trend_report.html (Content-Type: text/html)
- 텔레그램 발송:
· telegram_summary.txt 내용을 단일 메시지로 발송
· 이메일 발송 성공 여부와 무관하게 텔레그램은 항상 발송한다.
- 발송 결과 알림:
성공: '트렌드 보고서 발송 완료 | 이메일 {N}명 · 텔레그램'
실패: '발송 오류: {오류 내용}'



sns-trend-monitor 폴더에 sns-trend-workflow 워크플로를 만들어 줘.
파일은 반드시 .claude/commands/sns-trend-workflow.md로 만들어.
워크플로 (sns-trend-workflow):
1. keyword-collector: X·LinkedIn 키워드 수집
2. trend-analyzer: 전주 대비 변화 분석
3. report-composer: HTML 보고서 + 텔레그램 요약 작성
4. notifier: 이메일 + 텔레그램 동시 발송
수동 실행 트리거:
- '지금 트렌드 분석해 줘' 입력 시 즉시 텔레그램 메시지를 발송하라.



sns-trend-monitor 폴더에 run.bat 파일을 만들고,
Windows 작업 스케줄러에 SNSTrendMonitor-Weekly 이름으로 등록해 줘.
run.bat 파일 내용 (아래 코드 그대로 작성):
@echo off
setlocal
set "BASE_DIR=D:\claudecode\sns-trend-monitor"
set "LOG_DIR=%BASE_DIR%\logs"
set "LOG_FILE=%LOG_DIR%\run.log"
if not exist "%LOG_DIR%" mkdir "%LOG_DIR%"
echo. >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
echo [%date% %time%] SNSTrendMonitor-Weekly start >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
set "ANTHROPIC_API_KEY="
cd /d "%BASE_DIR%"
call claude -p "/sns-trend-workflow" --permission-mode bypassPermissions >> "%LOG_FILE%" 2>&1
echo [%date% %time%] SNSTrendMonitor-Weekly end (exit: %errorlevel%) >> "%LOG_FILE%"
endlocal
schtasks /create /tn "SNSTrendMonitor-Weekly" /tr "cmd /c D:\claudecode\sns-trend-monitor\run.
bat" /sc weekly /d mon /st 18:00 /f
등록 후 schtasks /query /tn \"SNSTrendMonitor-Weekly\"으로 확인해 줘.
run.bat 파일은 반드시 CRLF(Windows) 줄 끝 형식으로 저장해 줘.
