realestate-monitor 폴더에 watch_config.json 파일을 생성해 줘.
watch_config.json 내용:
- target_areas: ["서울 마포구 공덕동", "서울 강서구 마곡동"]
(감시할 지역 목록, 원하는 지역으로 수정 가능)
- property_types: ["아파트"]
- max_price_억: 20 (목표 매수 상한가, 억 단위)
- min_size_평: 25 (최소 면적)
- alert_score_threshold: 55 (미래가치 점수 기준, 100점 만점)
- check_interval_hours: 6 (6시간마다 자동 수집)
- telegram_bot_token: os.environ["TELEGRAM_BOT_TOKEN"] (환경 변수에서 읽어올 것)
- telegram_chat_id: os.environ["TELEGRAM_CHAT_ID"] (환경 변수에서 읽어올 것)
- telegram_alert: true
아울러 .env 파일을 생성하고 아래 내용을 채워 줘.
(독자는 나중에 실제 값으로 교체):
# 텔레그램 봇
TELEGRAM_BOT_TOKEN=여기에_BotFather에서_발급받은_토큰_입력
(예: 1234567890:ABCdefGHIjklMNOpqrSTUvwxYZ)
TELEGRAM_CHAT_ID=여기에_본인_채팅방_ID_입력
(확인 방법: 봇에 /start 전송 후
https://api.telegram.org/bot{토큰}/getUpdates에서 chat id 값 복사)
# 공공데이터포털 - 국토교통부 (아파트 매매 실거래가)
MOLIT_API_KEY=여기에_공공데이터포털_인증키_입력
MOLIT_API_ENDPOINT=여기에_data.go.kr에서_받은_End_Point_URL_입력
(확인 방법: data.go.kr 마이페이지 → 데이터활용 → 오픈 API →
활용신청 현황 → 해당 API 클릭 → "활용신청 상세보기" → End Point 값 복사)
(예: https://apis.data.go.kr/1613000/RTMSDataSvcAptTradeDev/getRTMSDataSvcAptTradeDev)
# 네이버 검색 API (실습 3.4에서 발급, 다른 실습과 공용 가능)
NAVER_CLIENT_ID=여기에_네이버_Client_ID_입력
NAVER_CLIENT_SECRET=여기에_네이버_Client_Secret_입력
# 네이버 부동산 (new.land.naver.com 개발자도구에서 복사)
NAVER_AUTH=여기에_네이버부동산_Authorization_헤더값_입력
("Bearer "로 시작하는 문자열 전체)
NAVER_COOKIE=여기에_네이버부동산_Cookie_헤더값_입력
(개발자 도구에서 복사한 cookie 값 전체)
# 나이스교육정보 (open.neis.go.kr 에서 발급)
NEIS_API_KEY=여기에_나이스교육정보_인증키_입력
.gitignore 파일도 생성해서 .env를 추가해 줘. (토큰이 Git에 노출되지 않도록)
기존 파일이 있으면 덮어 쓰지 말고 사용자에게 알려 줘.



realestate-monitor 폴더에 transport-analyzer 에이전트를 만들어 줘.
이 에이전트는:
- target_areas 각 지역의 교통 호재 뉴스를 수집한다.
출처: 국토교통부 보도자료 (molit.go.kr), 국가교통정보센터 (its.go.kr),
서울교통공사 (topis.seoul.go.kr), 각 지자체 도시계획 공고
- 키워드: 지하철 연장, GTX, 도로 개통, 버스 노선 신설, 트램
- 착공·개통 예정 시기와 예상 수혜 범위를 분석한다.
- 호재 점수를 0~30점으로 산출하여 transport_score.json에 저장한다.
(착공 완료: 25~30점, 계획 확정: 15~24점, 검토 중: 5~14점)



realestate-monitor 폴더에 school-info-collector 에이전트를 만들어 줘.
이 에이전트는:
- 각 지역의 초·중·고 배정 학교 정보를 수집한다.
출처: 교육부 학구도 안내 서비스 (schoolzone.moe.go.kr)
- 학교알리미(schoolinfo.go.kr) 기준 학업 성취도, 특목고 진학률 등을 파악한다.
(NEIS_API_KEY 필수: neis.go.kr에서 발급 후
.env에 NEIS_API_KEY=키값으로 추가해야 실제 데이터 수집 가능)
- 학군 점수를 0~20점으로 산출하여 school_score.json에 저장한다.
(강남 3구 수준: 18~20점, 준수: 12~17점, 평균: 6~11점)



realestate-monitor 폴더에 policy-monitor 에이전트를 만들어 줘.
이 에이전트는:
- 각 지역 관련 정부 정책 뉴스를 수집한다.
출처: 국토교통부 (molit.go.kr),
서울시 도시계획 포털 (news.seoul.go.kr/citybuild)
- 키워드: 재건축, 재개발, 용도지역 변경, 규제 완화, 토지거래허가구역
- 국토교통부·서울시 공식 발표와 네이버 뉴스 API(NAVER_CLIENT_ID/SECRET)를 사용해
동/구 단위 정책 뉴스를 수집한다
- 정책 영향도를 -10~+20점으로 산출하여 policy_score.json에 저장한다
(개발 호재: +10~+20점, 중립: 0점, 규제 강화: -10~-1점)



realestate-monitor 폴더에 value-analyzer 에이전트를 만들어 줘.
이 에이전트는:
- price_data.json, transport_score.json, school_score.json, policy_score.json을 읽는다
- 시세 대비 저평가 여부를 분석한다. (0~30점)
· 매물가가 평균 실거래가 대비 10% 이상 저렴하면 25~30점,
· 5~10% 저렴하면 15~24점, 평균 수준이면 5~14점
- 교통·학군·정책 점수를 합산하여 최종 미래가치 점수(100점 만점)를 산출한다.
최종 점수 = 시세점수(30) + 교통점수(30) + 학군점수(20) + 정책점수(20)
- 분석 결과를 value_report.json으로 저장한다.
- 점수와 함께 투자 의견(적극 매수 / 관망 / 보류)을 포함한다.
- 분석 완료 후 telegram-alert 에이전트를 호출한다.



realestate-monitor 폴더에 telegram-alert 에이전트를 만들어 줘.
이 에이전트는:
- value_report.json을 읽어 최종 점수를 확인한다.
- alert_score_threshold 이상인 매물이 있으면 텔레그램 메시지를 발송한다.
- 메시지 형식:
[매물 알람] 미래가치 {점수}점 매물 발견!
위치: {지역} | 면적: {평수}평 | 가격: {가격}억
분석 결과:
· 시세 대비: {평균가 대비 X% 저렴}
· 교통 호재: {교통 호재 요약}
· 학군 평가: {학군 등급}
· 정책 현황: {정책 영향 요약}
투자 의견: {적극 매수 / 관망 / 보류}
네이버 부동산 링크: {링크}



agents/price-collector.md 에이전트 정의를 만들어 줘.
역할: data.go.kr 국토교통부 실거래가 API로 watch_config.json의
target_areas 아파트 매매 가격을 수집해 price_data.json에 저장한다.
입력:
- .env: MOLIT_API_KEY, MOLIT_API_ENDPOINT
- watch_config.json: target_areas, property_types, min_size_평
산출물: price_data.json (지역별 단지별 최근 거래가)
규칙:
- 외부 패키지 금지, 표준 라이브러리(urllib)만 사용한다.
- serviceKey는 URL 인코딩 처리 (urllib.parse.quote)
- API 호출 실패 시 1회 재시도 → 실패 시 기존 price_data.json 유지하고
로그 남긴 뒤 다음 단계로 진행한다. (전체 중단 금지)
다른 에이전트(transport-analyzer, school-info-collector, policy-monitor,
value-analyzer, telegram-alert)와 동일한 markdown 포맷(name,
description, tools 필드 포함)으로 작성한다.
생성만 하고 이 단계에서는 단독 실행하지 마.
통합 테스트는 워크플로 단계(/property-monitor)에서 진행한다.



realestate-monitor 폴더에 property-monitor 워크플로를 만들어 줘.
파일은 반드시 .claude/commands/property-monitor.md로 만들어.
워크플로 (property-monitor):
1. price-collector로 주변 시세 수집
2. transport-analyzer로 교통 호재 분석
3. school-info-collector로 학군 정보 수집
4. policy-monitor로 정부 정책 모니터링
5. value-analyzer로 미래 가치 종합 분석
6. telegram-alert로 조건 충족 매물 알람 발송
수동 실행 트리거:
- '지금 매물 한 번 분석해 줘' 입력 시 즉시 실행한다.



realestate-monitor 폴더에 run.bat 파일을 만들고,
Windows 작업 스케줄러에 RealEstateMonitor-Daily 이름으로 등록해 줘.
run.bat 파일 내용 (아래 코드 그대로 작성):
@echo off
setlocal
set "BASE_DIR=D:\claudecode\realestate-monitor"
set "LOG_DIR=%BASE_DIR%\logs"
set "LOG_FILE=%LOG_DIR%\run.log"
if not exist "%LOG_DIR%" mkdir "%LOG_DIR%"
echo. >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
echo [%date% %time%] RealEstateMonitor-Daily start >> "%LOG_FILE%"
echo ================================================ >> "%LOG_FILE%"
set "ANTHROPIC_API_KEY="
cd /d "%BASE_DIR%"
call claude -p "지금 매물 한 번 분석해 줘" --permission-mode bypassPermissions >> "%LOG_FILE%"
2>&1
echo [%date% %time%] RealEstateMonitor-Daily end (exit: %errorlevel%) >> "%LOG_FILE%"
endlocal
schtasks /create /tn "RealEstateMonitor-Daily" /tr "cmd /c D:\claudecode\realestate-monitor\
run.bat" /sc daily /st 17:40 /f
등록 후 schtasks /query /tn \"RealEstateMonitor-Daily\"으로 확인해 줘.
run.bat 파일은 반드시 CRLF(Windows) 줄 끝 형식으로 저장해 줘.
